Introduction to CSS

Cascading Style Sheets (CSS) is a style sheet language used to change the styling/ presentation of a markup language such as HyperText Markup Language (HTML). By using CSS, you can change the layout, colours, fonts, sizes and many other features of a website. CSS was initially created on the 17th of December, 1996 by World Wide Web Consortium (W3C). When you create a CSS file, it is saved as a .css file, similar to how HTML is saved as a .html file.

What you should already know

This guide expects that you already know/have the following:

  • You should be relatively familiar with the basics of HTML. You can learn the required HTML knowledge here.
  • Basic experience with computers.
  • Basic networking knowledge (if you plan on publishing your website).

Basic CSS format

Similar to HTML, CSS is written in a certain format. When CSS is applied to an HTML document, it goes inside a <style> tag. Below is an example of the format that CSS is written in. 

                            
<style>
    The HTML element that you are styling will go here. {
    The CSS styling element will go here: The value of that styling element will go here;
    You can add more elements as well: You can add as many as you want;
    }
</style>
A semicolon is always required at the end of a CSS value, otherwise the browser won't process it.

Essential CSS elements

You will now learn the essential CSS elements and how you can incorporate them into your HTML documents. Below is a list that contains the essential elements. All of the elements listed below will use the format shown above. 

                            
<style>
    Here are all of the basic and essential CSS elements that you'll need to know {
        font-size: changes the size of the font;
        font-family: changes the font;
        color: changes the colour of the font;
        background-color: changes the colour of the background;
        background-image: sets the image of a background.
        text-align: changes the position of the text (left/right/centre/justify);
        border: creates a box around an element, but to sound very smart and professional it's called a border;
        padding: creates space around an element and its content inside a defined border;
        margin: the same thing as padding, but it creates space outside of a defined border;
        position: changes the positioning method of an element (static/relative/fixed/absolute/sticky);
        !important: prioritises this styling element over anything else. An example is included below;
        font-size: 18px !important;
    }
</style>
Congratulations! You now know the basic CSS elements.

Basic examples of CSS

In the example below, the styling elements of all h1 tags are being changed. 

                            
<style>
    h1 {
        font-size: 40px;
        font-family: font-family:Arial, Helvetica, sans-serif;
        color: white;
        background-color: rgb(0, 0, 0);
        /* ^ That means that the background colour will be black because black contains 0 red, 0 green and 0 blue. */
        padding: 5px;
        margin: 10px;
        position: relative;
    }
</style>
In the example below, the styling elements of the "main-doc-section" class are being changed. 
                            
<style>
    .main-doc-section {
        font-size: 18px;
        font-family: "Comic Sans MS", "Comic Sans", cursive;
        position: static;
        margin: 5px;
        padding: 10px;
        background image: url("example.png");
        color: rgb(5, 4, 10)
    }
</style>
In the example below, the styling elements of the "section-main" id are being changed. 
                            
<style>
    #section-main {
        font-size: 18px;
        font-family: "Comic Sans MS", "Comic Sans", cursive;
        position: static;
        margin: 5px;
        padding: 10px;
        background image: url("example.png");
        color: rgb(5, 4, 10)
    }
</style>
Those are some basic examples of CSS. Feel free to practice some CSS yourself!

CSS comments

You probably know that in HTML, comments are written like this: <!-- This is a comment --> (and if you didn't, you know it now.)
CSS also has a way of writing comments to make it easier for the user to navigate the .css document or to write whatever you want to. You may have noticed in one of the examples above that I wrote the following: /* ^ That means that the background colour will be black because black contains 0 red, 0 green and 0 blue. */ The /* */ characters represent a comment in CSS. After the first /* characters, you write whatever you would like to include in the comment. For this example, the comment will include the following text: "Lorem Ipsum." If you wanted to write a CSS comment that contains that text, the comment would look like this: /* Lorem Ipsum */
Congratulations, you now know how to write a comment in CSS!

Further learning

You now (hopefully) know the basics of CSS and how CSS works. However, you are no way near being a master in CSS. You still have lots to learn. If would like to learn more about CSS, which I would highly recommend if you have the time, you take the free responsive web design course on freeCodeCamp.org. This course teaches you a lot about web development and you will learn a lot about HTML and CSS by taking it. Once you have done the course and you feel like a genius, you will be presented with some challenges that will take away that feeling because you will have forgotten most of what you learnt. However, this is ok. Coding is not about knowing all the syntax instantly. Coding is about learning how things work, solving problems and finding patterns. During the challenges, if you are stuck (because you 100% will be), you can research your problem on W3Schools.com. W3Schools has information on every single HTML tag ever created and every single CSS element ever created, you can rely on that when you encounter problems.
